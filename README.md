# danalvarez-skills

A set of skills reflecting my style of programming.

## comment-style

I use this skill to make the agent comment code as I would. I prefer short comments that divide a function or piece of code into subsections. Short comments that make a function readable, where you can understand the overall logic being accomplished even if each independent line is not fully explained. I prefer comments to be on top of these subsections, instead of beside each line of code. And I always want code to be thoroughly documented with standard practices. For example, doxygen for C/C++, docstring for Python and roxygen for R. All functions should have a clear but brief explanation of what they do, what they accept and what they return.
